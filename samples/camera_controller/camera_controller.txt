サンプル: camera_controller

概要
- `EyeRig` の `base / rod / eye` 構成を前提に、`type` 切替で orbit / first-person / follow を同じ scene で比較するサンプルです。
- `WebgApp` の標準視点 rig を使いながらも、視点の本質が `Space.setEye(node)` であることを崩さず、helper ごとの差だけを見られる構成にしています。
- 追従対象ノードの移動、first-person の手動移動、orbit の全景確認を 1 つの画面で見比べ、`base / rod / eye` のどこが動いているかを HUD で追えるようにしています。

このサンプルで使うwebg機能
- `WebgApp`: 起動、描画ループ、HUD 表示、入力 lower-case 規約の共通化
- `Shape` / `Node`: 床、塔、ターゲットキャラクタ、進行方向マーカーの構築
- `Message`: 画面上の操作ガイド表示

確認ポイント
- orbit では `rod` が主に回り、first-person では `base` と `eye` が主に働き、follow では target 追従により `base` が更新されることを HUD の `base / rod / eye` 表示で確認します。
- follow helper で target の向きに応じて後方視点が保たれ、target が円軌道を回っても視点が破綻しないことを確認します。
- orbit / follow では touch の 1本指 drag で回転、2本指 drag で平行移動、pinch で距離調整でき、mouse と同じ scene を mobile でも見回せることを確認します。
- `EyeRig` により、進行方向と少し違う方向を見るための独立視線を後から追加しやすい構造になっていることを確認します。

操作説明
- `1`: orbit view へ切替
- `2`: first-person view へ切替
- `3`: follow view へ切替
- `P`: moving target の移動を一時停止 / 再開
- `R`: 現在の view mode の姿勢を初期値へ戻す
- orbit / follow: ドラッグまたは `Arrow` キーで回転
- orbit / follow: 2本指ドラッグで平行移動
- orbit / follow: pinch、ホイール、または `[` / `]` で距離調整
- first-person: ドラッグで視線回転
- first-person: `W` / `A` / `S` / `D` で移動、`Q` / `E` で下降 / 上昇、`Shift` で加速
