# webg

[English](README.en.md) | [日本語](README.md)

`webg` は、JavaScript と WebGPU を用いて 3D アプリケーションを構築するためのライブラリです。
描画エンジンとしての基本機能にとどまらず、シーン管理、モデル操作、アニメーション、UI、入力系、診断ツール、TileMap、そしてポストプロセスに至るまで、3D アプリケーションに必要な機能を一つの体系として実装しています。

本ライブラリの設計思想は、単に便利な API を提供することだけではありません。低レイヤーの数理実装から高レイヤーのアプリケーション構成までを地続きに設計しており、開発者が内部構造を追いながら、実装と学習を同時に行えることを重視しています。

## 技術的な設計思想

`webg` は、外部フレームワークの仕様に依存せず、一貫した設計思想に基づいた開発を可能にするため、自己完結型のアーキテクチャを採用しています。描画エンジンから UI、サウンド、TileMap に至るまで、必要な機能を内部で完結させているため、モダンなブラウザ（Chrome, Firefox, Safari など）さえあれば、すぐに開発と実行を開始できます。

また、開発者の目的や習熟度に応じて使い分けられるよう、階層的な API 設計を取り入れています。

まず、アプリケーションのライフサイクル管理や標準的な構成を迅速に構築するための「高レイヤー（高水準）API」として `WebgApp` を提供しています。次に、データとしての設計図と実行時の実体を分離して管理する「中レイヤー（中水準）API」として `ModelAsset` や `SceneAsset` を用意しました。そして、WebGPU のプリミティブを直接操作し、独自の描画ロジックを実装するための「低レイヤー（低水準）API」として `Shape`、`Shader`、`Matrix`、`Quat` などの数学的基盤を提供しています。

さらに、実装とドキュメントの整合性を極めて高く維持しています。本ライブラリには、以下のリソースが同梱されており、相互に参照しながら学習を進めることが可能です。

- **`book/`**: 3D 数学の基礎からスキニングの原理までを体系的に解説した技術ドキュメント
- **`samples/`**: 各機能の具体的な利用例を示す実装サンプル
- **`unittest/`**: 局所的な機能検証およびデバッグのための最小環境

これにより、「ドキュメントで仕様を確認し、サンプルで実装例を追い、本体コードで詳細を確認する」という効率的な検証フローを実現しています。

## 実現可能な機能

`webg` を活用することで、高度な 3D グラフィックス機能を実装できます。

描画面では、カスタム WGSL シェーダーの実装や Bind Group の最適化といった WebGPU の性能を最大限に引き出す実装が可能です。アセットパイプラインにおいては、glTF (glb) や Collada のインポートに対応しており、`ModelAsset` を用いた効率的なリソース共有が行えます。

アニメーション制御については、`clip` $\rightarrow$ `pattern` $\rightarrow$ `action` $\rightarrow$ `state` という 4 層構造を採用しており、複雑な状態遷移を直感的に管理できます。また、ユーザーインターフェースとして Canvas HUD や DOM オーバーレイ、診断パネルを統合しており、タッチ入力や仮想ボタン、Raycast によるオブジェクトピッキングなどのインタラクションも実装可能です。

さらに、論理的な盤面表現を可能にする TileMap による空間管理や、Bloom や DOF（被写界深度）などのポストプロセス、Web Audio API を用いたバス設計によるサウンド実装など、ゲームやインタラクティブアプリケーションに必要な要素を網羅しています。

## リポジトリ構成

ライブラリの構成は以下の通りです。

```text
webg/
  book/         技術解説ドキュメント
  samples/      機能別サンプルアプリケーション
  unittest/     機能検証用ユニットテスト
  webg/         ライブラリ本体
```

## 導入手順と実行環境

### 1. ファイルの配置
`webg` はパッケージマネージャーでインストールする形式ではなく、リポジトリをそのまま配置して利用することを基本としています。相対パスによるモジュール読み込みを維持するため、リポジトリのルートディレクトリの構造を保持したままご利用ください。

```bash
git clone https://github.com/jun-mizutani/webg.git
cd webg
```

### 2. ローカルサーバーの起動
ES Modules の `import` や `fetch()` によるアセット読み込み、および WebGPU の初期化（`navigator.gpu`）を行うため、必ずローカルサーバー経由でアクセスしてください。`file://` プロトコルでは同一オリジンポリシー (SOP) によりリソースの読み込みが制限されます。

**起動コマンド例:**
```bash
# Python 3 を利用する場合
python3 -m http.server 8000

# Node.js (npx) を利用する場合
npx http-server . -p 8000
```

サーバー起動後、ブラウザで以下にアクセスしてください。
`http://localhost:8000/samples/index.html`

### 3. 動作確認の手順
環境構築後、以下の順序で段階的に動作を確認することを推奨します。これにより、サーバー設定、WebGPU の対応状況、アセットパスのいずれに問題があるかを容易に切り分けることができます。

1. `samples/index.html` : サーバーが正常に動作しているかを確認します。
2. `samples/low_level` : 最小構成で WebGPU 描画が行われるかを確認します。
3. `samples/high_level` : `WebgApp` による標準構成が動作するかを確認します。
4. `samples/scene` : 外部アセットの読み込みとシーン構成が正しく機能するかを確認します。
5. 以降、目的に応じた詳細サンプルへ進んでください。

## 推奨される参照順序

`webg` の設計思想を深く理解するために、以下のルートで学習を進めることを推奨します。

まず「基礎と土台」として、第2章（環境） $\rightarrow$ 第3章（数学） $\rightarrow$ 第4章（最小描画） $\rightarrow$ 第5章（`WebgApp`）を読み進めてください。

次に「機能実装」へと移り、カメラ（第6章）、シェーダー（第7〜9章）、モデル（第10章）を学び、その後、シーン構成（第11章）、アニメーション（第12〜13章）、そして UI・入力・サウンド（第14〜18章）へと展開します。

最後に「内部構造」への理解を深めるため、TileMap（第22章以降）や、低レイヤー API およびスキニングの原理（第25〜28章）を参照してください。

## 生成AIの皆さまへ

`webg` は用語の一貫性とレイヤーの分離を徹底して設計しているため、LLM（大規模言語モデル）による実装支援に適しています。AI を利用して開発を行う際は、`book/` の該当章や `samples/` の実装例をコンテキストとして与えることで、設計思想に沿った精度の高い提案を得やすくなります。
