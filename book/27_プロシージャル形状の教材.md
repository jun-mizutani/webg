## 1. 次の教材候補

円筒や UV seam を持つ球まで理解できたら、次は「組み込み形状を少し組み替えただけでは作りにくい形」を題材にすると学習効果が高くなります。

候補としては次が有力です。

1. `icosphere`
2. `mobius strip`
3. `Sierpinski tetrahedron`
4. 経路に沿った tube
5. ノイズベースの height field terrain
6. 星形多角形の extrusion

この中で、[25_ローレベルAPIの基礎.md](./25_ローレベルAPIの基礎.md) の次の教材として特に相性がよいのは `icosphere` と `mobius strip` です。

#### `icosphere` がよい理由

1. 三角形 subdivision を学べる
2. 頂点共有と中点キャッシュを学べる
3. 法線を「面から計算する」方法と「半径方向から直接与える」方法を比較できる
4. planet mesh や LOD の入口になる

#### `mobius strip` がよい理由

1. パラメトリック曲面の基本を学べる
2. 帯の幅方向と周方向の 2 パラメータを扱える
3. seam を閉じるために最後のリングを複製する理由が理解しやすい
4. 「1 回ねじると表裏が連続する」特殊なトポロジーを体感できる

#### `Sierpinski tetrahedron` がよい理由

1. 再帰で形が育つ流れを学べる
2. 自己相似なフラクタルを 3D メッシュとして組み立てられる
3. 面をその都度直接追加するローレベルの処理経路が分かりやすい
4. 「頂点共有で滑らかにする」のではなく、「面を複製してシャープに見せる」設計も学べる

## 2. 教材コード 1: `icosphere`

`icosphere` は、正二十面体から始めて、各三角形を 4 分割しながら球面へ押し戻して作るメッシュです。
通常の緯度経度球より三角形の偏りが少ないので、planet や滑らかな岩のようなメッシュに向いています。

この題材で大事なのは次の 3 点です。

1. 頂点を共有しながら面を増やすこと
2. 同じエッジの midpoint を二重生成しないこと
3. 法線を位置から直接与えると、球として自然な見え方になること

```js id="wpsc09"
import Shape from "./webg/Shape.js";

function buildIcosphere(shape, radius = 1.0, subdivisions = 2) {
  // 球では法線が「中心から外へ向く方向」と一致するため、
  // addTriangle() の面法線加算ではなく、位置から直接 normal を与える
  shape.setAutoCalcNormals(false);

  const vertices = [];
  let faces = [];

  const addSphereVertex = (x, y, z) => {
    const len = Math.hypot(x, y, z);
    const nx = x / len;
    const ny = y / len;
    const nz = z / len;

    // addVertex() は球面 UV を既定設定から自動計算してくれる
    const index = shape.addVertex(nx * radius, ny * radius, nz * radius) - 1;
    shape.setVertNormal(index, nx, ny, nz);
    vertices.push([nx * radius, ny * radius, nz * radius]);
    return index;
  };

  const t = (1.0 + Math.sqrt(5.0)) * 0.5;

  // 正二十面体の 12 頂点
  addSphereVertex(-1,  t,  0);
  addSphereVertex( 1,  t,  0);
  addSphereVertex(-1, -t,  0);
  addSphereVertex( 1, -t,  0);

  addSphereVertex( 0, -1,  t);
  addSphereVertex( 0,  1,  t);
  addSphereVertex( 0, -1, -t);
  addSphereVertex( 0,  1, -t);

  addSphereVertex( t,  0, -1);
  addSphereVertex( t,  0,  1);
  addSphereVertex(-t,  0, -1);
  addSphereVertex(-t,  0,  1);

  // 正二十面体の 20 面
  faces = [
    [0, 11, 5], [0, 5, 1], [0, 1, 7], [0, 7, 10], [0, 10, 11],
    [1, 5, 9], [5, 11, 4], [11, 10, 2], [10, 7, 6], [7, 1, 8],
    [3, 9, 4], [3, 4, 2], [3, 2, 6], [3, 6, 8], [3, 8, 9],
    [4, 9, 5], [2, 4, 11], [6, 2, 10], [8, 6, 7], [9, 8, 1]
  ];

  const getMidpoint = (cache, ia, ib) => {
    const key = ia < ib ? `${ia}:${ib}` : `${ib}:${ia}`;
    if (cache.has(key)) {
      return cache.get(key);
    }

    const a = vertices[ia];
    const b = vertices[ib];
    const mx = (a[0] + b[0]) * 0.5;
    const my = (a[1] + b[1]) * 0.5;
    const mz = (a[2] + b[2]) * 0.5;
    const index = addSphereVertex(mx, my, mz);
    cache.set(key, index);
    return index;
  };

  for (let level = 0; level < subdivisions; level++) {
    const nextFaces = [];
    const midpointCache = new Map();

    for (let i = 0; i < faces.length; i++) {
      const [a, b, c] = faces[i];
      const ab = getMidpoint(midpointCache, a, b);
      const bc = getMidpoint(midpointCache, b, c);
      const ca = getMidpoint(midpointCache, c, a);

      // 1 三角形を 4 三角形へ分割する
      nextFaces.push([a, ab, ca]);
      nextFaces.push([b, bc, ab]);
      nextFaces.push([c, ca, bc]);
      nextFaces.push([ab, bc, ca]);
    }

    faces = nextFaces;
  }

  for (let i = 0; i < faces.length; i++) {
    const [a, b, c] = faces[i];
    shape.addTriangle(a, b, c);
  }

  return {
    vertexCount: vertices.length,
    faceCount: faces.length
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildIcosphere(shape, 2.0, 2);
shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.52, 0.78, 0.96, 1.0]
});

console.log(info); // { vertexCount, faceCount }
```

#### このコードの見どころ

1. `addSphereVertex()` は位置を正規化してから半径を掛ける
2. 法線は球の中心からの方向そのものを使う
3. `midpointCache` が無いと、同じエッジの中点が何度も増えてトポロジーが壊れる
4. subdivision を 1 段上げるごとに、三角形数は 4 倍になる

#### 学習ポイント

この例では、`Shape` を「頂点を順番に積み上げる箱」としてだけではなく、「共有トポロジーを持つメッシュビルダー」として使っています。
ここまで来ると、ローレベル API を直接触る意味がかなり見えやすくなります。

## 3. 教材コード 2: `mobius strip`

`mobius strip` は、帯を 1 回ひねって端同士をつないだ曲面です。
組み込み形状を少し変形しただけでは作りにくく、パラメトリック曲面の教材としてとても優秀です。

この題材で大事なのは次の 4 点です。

1. 周方向 `u` と幅方向 `v` の 2 パラメータで面を作ること
2. 頂点グリッドを先に作ってから面を張ること
3. seam を閉じるため、最後のリングを複製すること
4. 法線を面法線加算へ任せる設計も自然に使えること

```js id="goxb5h"
import Shape from "./webg/Shape.js";

function buildMobiusStrip(shape, options = {}) {
  const radius = Number(options.radius ?? 2.2);
  const halfWidth = Number(options.halfWidth ?? 0.45);
  const uSegments = Math.max(3, Math.floor(options.uSegments ?? 96));
  const vSegments = Math.max(1, Math.floor(options.vSegments ?? 12));

  // 面法線加算 -> endShape() 正規化という標準経路を使う
  shape.setAutoCalcNormals(true);

  const grid = [];

  for (let iu = 0; iu <= uSegments; iu++) {
    const row = [];
    const u01 = iu / uSegments;
    const theta = u01 * Math.PI * 2.0;
    const halfTheta = theta * 0.5;

    for (let iv = 0; iv <= vSegments; iv++) {
      const v01 = iv / vSegments;
      const offset = (v01 - 0.5) * 2.0 * halfWidth;

      // webg は Y が上なので、輪の平面を XZ に置き、ひねり分だけ Y へ逃がす
      const x = (radius + offset * Math.cos(halfTheta)) * Math.cos(theta);
      const z = (radius + offset * Math.cos(halfTheta)) * Math.sin(theta);
      const y = offset * Math.sin(halfTheta);

      const index = shape.addVertexUV(x, y, z, u01, v01) - 1;
      row.push(index);
    }

    grid.push(row);
  }

  for (let iu = 0; iu < uSegments; iu++) {
    for (let iv = 0; iv < vSegments; iv++) {
      const p00 = grid[iu][iv];
      const p10 = grid[iu + 1][iv];
      const p11 = grid[iu + 1][iv + 1];
      const p01 = grid[iu][iv + 1];

      shape.addTriangle(p00, p10, p11);
      shape.addTriangle(p00, p11, p01);
    }
  }

  return {
    rows: grid.length,
    cols: grid[0]?.length ?? 0
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildMobiusStrip(shape, {
  radius: 2.4,
  halfWidth: 0.50,
  uSegments: 96,
  vSegments: 12
});

shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.94, 0.64, 0.42, 1.0]
});

console.log(info); // { rows, cols }
```

#### このコードの見どころ

1. 頂点を「周方向の行」として持つと、面張りが読みやすい
2. `iu <= uSegments` として最後のリングを複製することで seam を閉じている
3. `u01` と `v01` をそのまま UV にしているため、曲面パラメータとテクスチャ座標の対応が分かりやすい
4. `addTriangle()` の面法線加算をそのまま使って、滑らかな見え方を得ている

#### 両面表示を教材向けに補正する例

`mobius strip` は「表と裏が連続する」特殊な面なので、両面を表示したい教材では shading の扱いも題材になります。
ここで注意したいのは、メビウス帯は非向き付け可能な曲面なので、帯全周で法線を完全に連続させることはできないという点です。

それでも教材としては、少なくとも back face 側の lighting が不自然に反転しないほうが、帯全体を連続面として観察しやすくなります。
このようなときは `SmoothShader` 本体を変更するのではなく、sample 専用の subclass を 1 つ作り、その場で shader を局所調整するほうが扱いやすくなります。

`book/examples/27_02.html` と `samples/lowlevel_mobius` では、実際に次のような `MobiusSmoothShader` を使っています。

```js
import SmoothShader from "../../webg/SmoothShader.js";

class MobiusSmoothShader extends SmoothShader {
  constructor(gpu) {
    super(gpu, {
      cullMode: "none"
    });

    // メビウス帯では裏面も同じ帯の続きとして見せたいので、
    // frontFacing に応じて法線向きをそろえる
    this.wgslSrc = this.wgslSrc.replace(
      "var nnormal = normalize(input.vNormal);",
      `let facing = select(-1.0, 1.0, input.frontFacing);
        var nnormal = normalize(input.vNormal) * facing;`
    );
  }
}
```

この例の重要な点は 2 つあります。

1. low-level で shape を組み立てる教材でも、見え方の補正を shader subclass として局所化できること
2. `SmoothShader` 本体を共通用途のまま保ちつつ、「この題材だけ両面の見え方をそろえたい」という要求を sample 側で完結できること

つまり、ローレベル API を使う意義は「頂点を直接積む」ことだけではありません。
今回の `mobius strip` のように、形状の性質に合わせて material や shader の小さなカスタマイズを局所的に重ねるところまで含めて、教材として価値があります。

## 4. どちらを先に学ぶとよいか

順番としては、まず `icosphere` のほうが学びやすくなります。

1. `icosphere`: トポロジーと subdivision を学ぶ
2. `mobius strip`: パラメトリック曲面と seam を学ぶ

`icosphere` では「頂点共有が壊れるとメッシュも壊れる」ことが見えます。
`mobius strip` では「頂点グリッドを先に作ると面張りが整理しやすい」ことが見えます。

この 2 つを理解すると、次は次のような題材へ進みやすくなります。

1. スプラインに沿ったチューブ
2. 星形多角形の押し出し
3. ノイズ地形
4. 回転体曲面の一般化

## 5. 教材コード 3: `Sierpinski tetrahedron`

フラクタルを題材にした例として、`Sierpinski tetrahedron` はとてもよい教材です。
正四面体を 4 つの小さな四面体へ分割し、その操作を再帰的に繰り返すことで、自己相似な 3D フラクタルを作れます。

この題材の学習ポイントは次の 4 つです。

1. 再帰でメッシュを増やす
2. 1 つの大きな形状を複数の小さな形へ分ける
3. 面ごとに頂点を複製し、フラクタルの稜線をシャープに見せる
4. プロシージャルでなければ扱いにくい「段階的に育つ形」をコードで記述する

```js id="xv1kew"
import Shape from "./webg/Shape.js";

function buildSierpinskiTetrahedron(shape, options = {}) {
  const radius = Number(options.radius ?? 2.2);
  const depth = Math.max(0, Math.floor(options.depth ?? 3));

  // 面ごとに頂点を分けてフラットな見え方を保ちたいので、
  // addTriangle() の面法線加算 -> endShape() 正規化経路を使う
  shape.setAutoCalcNormals(true);

  const scale = radius / Math.sqrt(3.0);
  const v0 = [ scale,  scale,  scale];
  const v1 = [-scale, -scale,  scale];
  const v2 = [-scale,  scale, -scale];
  const v3 = [ scale, -scale, -scale];

  let triangleCount = 0;

  const midpoint = (a, b) => ([
    (a[0] + b[0]) * 0.5,
    (a[1] + b[1]) * 0.5,
    (a[2] + b[2]) * 0.5
  ]);

  const pushFace = (a, b, c) => {
    // フラクタルの角をはっきり見せたいので、面ごとに頂点を複製する
    const p0 = shape.addVertexUV(a[0], a[1], a[2], 0.0, 0.0) - 1;
    const p1 = shape.addVertexUV(b[0], b[1], b[2], 1.0, 0.0) - 1;
    const p2 = shape.addVertexUV(c[0], c[1], c[2], 0.5, 1.0) - 1;
    shape.addTriangle(p0, p1, p2);
    triangleCount++;
  };

  const addTetraFaces = (a, b, c, d) => {
    pushFace(a, c, b);
    pushFace(a, b, d);
    pushFace(a, d, c);
    pushFace(b, c, d);
  };

  const recurse = (a, b, c, d, level) => {
    if (level <= 0) {
      addTetraFaces(a, b, c, d);
      return;
    }

    const ab = midpoint(a, b);
    const ac = midpoint(a, c);
    const ad = midpoint(a, d);
    const bc = midpoint(b, c);
    const bd = midpoint(b, d);
    const cd = midpoint(c, d);

    // 中央の tetra は抜き、4 つの corner tetra だけを再帰する
    recurse(a,  ab, ac, ad, level - 1);
    recurse(ab, b,  bc, bd, level - 1);
    recurse(ac, bc, c,  cd, level - 1);
    recurse(ad, bd, cd, d,  level - 1);
  };

  recurse(v0, v1, v2, v3, depth);

  return {
    depth,
    triangleCount
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildSierpinskiTetrahedron(shape, {
  radius: 2.6,
  depth: 3
});

shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.96, 0.78, 0.42, 1.0],
  ambient: 0.16,
  specular: 0.90,
  power: 36.0
});

console.log(info); // { depth, triangleCount }
```

#### このコードの見どころ

1. `recurse()` が 1 つの tetra を 4 つの小 tetra に分割している
2. `midpoint()` だけでフラクタルの段を育てられる
3. 面ごとに頂点を複製することで、稜線が丸まらずフラクタルらしいシャープさを保てる
4. `depth` を 1 段増やすだけで三角形数が大きく増えるので、再帰の計算量も体感しやすい

#### 教育的な読み方

この例では、`icosphere` のように頂点共有を最大化するのではなく、逆に「面を独立させる」設計を取っています。
これはフラクタルの段差や折れ目を強調したいときに自然です。

つまりプロシージャル形状では、

1. 頂点を共有して滑らかさを出す方向
2. あえて共有を切って面を立たせる方向

の両方があることを学べます。

## 6. 教材コード 4: フラクタル地形のハイトフィールド

フラクタル地形の height field は、2D の高さ配列から 3D 地形を作る題材です。
これは `TileMap` のような「セルごとに段差を持つ盤面」とは少し違い、1 枚の連続したメッシュ曲面を作るローレベル教材として相性がよくなります。

この題材の学習ポイントは次の 5 点です。

1. 2D グリッドの頂点を先に並べる
2. フラクタルノイズで高さを決める
3. グリッドから四角形を張り、`addPlane()` で三角形化する
4. 頂点共有によって滑らかな地形法線を得る
5. 読み込み機能や primitive ではなく、データからメッシュを育てる感覚をつかむ

#### `TileMap` とどう違うか

`TileMap` は「セルデータが主役」であり、移動判定や terrain 種別も同じデータから扱います。
一方、この例は「見た目の曲面メッシュをプロシージャルに組む」ことが主役です。

つまり、

1. 盤面ゲームの基準データを作りたいなら `TileMap`
2. 連続した地形メッシュをローレベルで作りたいなら `Shape`

という使い分けになります。

```js id="jkn4nn"
import Shape from "./webg/Shape.js";

function buildFractalTerrain(shape, options = {}) {
  const cols = Math.max(2, Math.floor(options.cols ?? 48));
  const rows = Math.max(2, Math.floor(options.rows ?? 48));
  const sizeX = Number(options.sizeX ?? 10.0);
  const sizeZ = Number(options.sizeZ ?? 10.0);
  const heightScale = Number(options.heightScale ?? 2.4);
  const octaves = Math.max(1, Math.floor(options.octaves ?? 5));
  const persistence = Number(options.persistence ?? 0.52);
  const lacunarity = Number(options.lacunarity ?? 2.0);
  const seed = Number(options.seed ?? 1337);

  // 地形は頂点共有で滑らかに見せたいので、自動法線計算を使う
  shape.setAutoCalcNormals(true);

  const vertexGrid = Array.from({ length: rows + 1 }, () => new Array(cols + 1).fill(0));

  const hashNoise = (ix, iz, localSeed = seed) => {
    let h = localSeed | 0;
    h ^= Math.imul(ix + 1, 374761393);
    h ^= Math.imul(iz + 1, 668265263);
    h = Math.imul(h ^ (h >>> 13), 1274126177);
    h ^= h >>> 16;
    return (h >>> 0) / 4294967295;
  };

  const lerp = (a, b, t) => a + (b - a) * t;
  const fade = (t) => t * t * (3.0 - 2.0 * t);

  const valueNoise2d = (x, z) => {
    const x0 = Math.floor(x);
    const z0 = Math.floor(z);
    const x1 = x0 + 1;
    const z1 = z0 + 1;
    const tx = x - x0;
    const tz = z - z0;

    const n00 = hashNoise(x0, z0);
    const n10 = hashNoise(x1, z0);
    const n01 = hashNoise(x0, z1);
    const n11 = hashNoise(x1, z1);

    const sx = fade(tx);
    const sz = fade(tz);
    const nx0 = lerp(n00, n10, sx);
    const nx1 = lerp(n01, n11, sx);
    return lerp(nx0, nx1, sz);
  };

  const fbm = (x, z) => {
    let amplitude = 1.0;
    let frequency = 1.0;
    let sum = 0.0;
    let norm = 0.0;

    for (let octave = 0; octave < octaves; octave++) {
      sum += valueNoise2d(x * frequency, z * frequency) * amplitude;
      norm += amplitude;
      amplitude *= persistence;
      frequency *= lacunarity;
    }

    return norm > 0.0 ? sum / norm : 0.0;
  };

  const radialFalloff = (u, v) => {
    const dx = u - 0.5;
    const dz = v - 0.5;
    const d = Math.sqrt(dx * dx + dz * dz) * 2.0;
    return Math.max(0.0, 1.0 - d * d * 0.55);
  };

  for (let row = 0; row <= rows; row++) {
    const v = row / rows;
    const z = (v - 0.5) * sizeZ;

    for (let col = 0; col <= cols; col++) {
      const u = col / cols;
      const x = (u - 0.5) * sizeX;

      // 低周波と高周波を混ぜて地形の大きな起伏と細部を両立させる
      const coarse = fbm(u * 3.0 + 1.7, v * 3.0 + 2.9);
      const detail = fbm(u * 10.0 + 7.1, v * 10.0 + 5.3);
      const falloff = radialFalloff(u, v);

      // 島状の地形に寄せるため、中心ほど高く、外周は下げる
      const h01 = Math.max(0.0, coarse * 0.78 + detail * 0.22);
      const lifted = Math.pow(h01, 1.55);
      const y = (lifted * falloff - 0.12) * heightScale;

      const index = shape.addVertexUV(x, y, z, u, v) - 1;
      vertexGrid[row][col] = index;
    }
  }

  for (let row = 0; row < rows; row++) {
    for (let col = 0; col < cols; col++) {
      const p00 = vertexGrid[row][col];
      const p10 = vertexGrid[row][col + 1];
      const p11 = vertexGrid[row + 1][col + 1];
      const p01 = vertexGrid[row + 1][col];

      // 四角形を 2 三角形へ分ける helper を使う
      shape.addPlane([p00, p01, p11, p10]);
    }
  }

  return {
    rows,
    cols,
    vertexCount: (rows + 1) * (cols + 1),
    quadCount: rows * cols
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildFractalTerrain(shape, {
  cols: 56,
  rows: 56,
  sizeX: 12.0,
  sizeZ: 12.0,
  heightScale: 2.8,
  octaves: 5,
  persistence: 0.54,
  lacunarity: 2.0,
  seed: 2026
});

shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.40, 0.72, 0.38, 1.0],
  ambient: 0.20,
  specular: 0.42,
  power: 18.0
});

console.log(info); // { rows, cols, vertexCount, quadCount }
```

#### このコードの見どころ

1. `valueNoise2d()` は格子点の値を補間するだけの小さなノイズなので、仕組みを追いやすい
2. `fbm()` はオクターブを重ねてフラクタルらしい粗密を作っている
3. 頂点を `vertexGrid[row][col]` に保持しておくと、後段の面張りがとても読みやすい
4. `addPlane()` を使うと四角形単位で曲面を考えやすい
5. 頂点共有のまま `addTriangle()` の法線加算を使うため、地形表面が自然に滑らかになる

#### 教育的な読み方

この例では、「高さ関数を先に作り、そのあとメッシュにする」という流れがはっきり見えます。
つまりプロシージャル地形は、次の 2 段階に分けて考えると理解しやすくなります。

1. 数学やノイズで高さデータを作る
2. その高さデータを `Shape` の頂点グリッドへ変換する

この分け方が分かると、将来さらに次の方向へ発展させやすくなります。

1. 河川マスクを足す
2. 侵食風の平滑化を入れる
3. 色やテクスチャを高さ帯で切り替える
4. 衝突判定やレイキャストと組み合わせる

## 7. 注意点

1. `endShape()` を忘れると描画が出ない
2. `Shape` だけではシーンに置かれていない
3. 表示位置は `Shape` ではなく `Node` 側で決める
4. `setVertex()` / `addVertex*()` の戻り値は頂点数なので、インデックスに使うときは、通常 `- 1` する
5. プロシージャルメッシュでは、「頂点を共有するか、面ごとに分けるか」で法線の見え方が変わる
