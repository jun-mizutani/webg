# 20 ポストプロセス

最終更新：2026-04-08 JST

本ガイドは、webg におけるポストプロセス処理を「いつ使い、どこへ組み込み、毎フレーム何を呼び出すか」が明確に分かる形で整理したものです。対象となるファイルは `webg/BloomPass.js`、`webg/DofPass.js`、`webg/VignettePass.js`、`webg/RenderTarget.js`、`webg/FullscreenPass.js`、`webg/SeparableBlurPass.js`、`webg/samples/bloom/main.js`、`webg/samples/dof/main.js`、`webg/unittest/vignette/main.js` です。画面全体の処理に続いて、scene 内の局所的な演出を軽量に足したい場合は、次章の [21_パーティクルと軽量エフェクト.md](./21_パーティクルと軽量エフェクト.md) を続けて読むと違いが整理しやすくなります。

背景として、ポストプロセス処理は 3D シーンのシェーダーを直接書き換える機能ではありません。まずシーンをオフスクリーンへ描画し、その結果のカラーテクスチャや深度テクスチャを後段のフルスクリーンパスで読み取り、最後にキャンバスへ戻します。この流れを理解せずに導入すると、「BloomPass を実装したのに画面が変化しない」「被写界深度（DOF）を実装したのに深度が読み取れない」「Vignette を呼び出したのにシーンと HUD の順番が崩れる」といった課題が発生しやすくなります。この章では、まずその流れをそろえることから始めます。

## この章で押さえること

### A. ポストプロセスはシーン描画の内部ではなく、シーン描画の後段に挟む別工程

`BloomPass` も `DofPass` も `VignettePass` も、「3D シーンを描き終えたあとのテクスチャ」を加工する部品です。3D オブジェクトのシェーダーや `Space.draw()` の中へ直接混ざるわけではありません。したがって導入するときは、まず「シーンをどこへ描くか」をオフスクリーンターゲットへ切り替える必要があります。

### B. 最初に変えるのはシェーダーではなく描画フロー

ポストプロセスを入れるときに最初に変えるのは `Phong` や `NormPhong` ではなく、`WebgApp.start()` の描画フローです。`autoDrawScene: false` にして、`onBeforeDraw` でオフスクリーンへ 3D シーンを描き、`onAfterDraw3d` でポストプロセスをキャンバスへ戻す形にすると整理しやすくなります。

### C. `BloomPass` と `DofPass` は「beginScene() -> space.draw() -> render()」で使う

この 2 つは流れがほぼ同じです。

1. `beginScene()` でオフスクリーンターゲットを開く
2. `app.space.draw(app.eye)` で 3D シーンをそのターゲットへ描画する
3. `render()` でぼかしや合成を実行してキャンバスへ戻す

この 3 段を 1 フレーム内で崩さないことが、最も重要です。

### D. 被写界深度（DOF）は `sampleDepth: true` が必要

Bloom はカラーだけを読めば動作しますが、被写界深度（DOF）は深度も後段シェーダーから読みます。したがって `DofPass` の内部では `RenderTarget({ sampleDepth: true })` が使われています。自分で同じ構成を組むときも、「深度を後段で読むなら、サンプリング可能な深度ターゲットが必要」という前提を外さないほうが安全です。

### E. HUD はポストプロセスの後に重ねる

ポストプロセス後も `Message` やデバッグオーバーレイを重ねたい場合は、最終パスのあとで深度をクリアし直す構成が必要です。`webg/samples/bloom` と `webg/samples/dof` は `app.screen.clearDepthBuffer()` をポストプロセスのあとに呼んでいます。これはエフェクトそのものではなく、描画フローを崩さないための処理です。

## この章の流れ

webg のポストプロセス処理を使うときは、次の順序で考えると迷いにくくなります。

1. `WebgApp` を `autoDrawScene: false` で起動します
2. `BloomPass`、`DofPass`、`VignettePass` のどれを使うか決めて生成します
3. 必要ならデバッグ用の `FullscreenPass` も一緒に作ります
4. `onUpdate()` でカメラやパラメータを更新します
5. `onBeforeDraw()` でポストプロセス側のオフスクリーンターゲットを開き、`app.space.draw(app.eye)` を呼びます
6. `onAfterDraw3d()` で `render()` を呼んでキャンバスへ戻します
7. デバッグビューを見せたいときは、`FullscreenPass.draw(target)` で中間テクスチャを表示します
8. 最後に HUD を重ねるなら `app.screen.clearDepthBuffer()` を呼びます

使い分けは、次の理解で十分です。

1. 光っている部分だけをにじませたいなら `BloomPass`
2. 距離に応じて手前や奥をぼかしたいなら `DofPass`
3. 最終色の周辺だけを暗くしたいなら `VignettePass`
4. 中間結果をそのまま画面へ出したいだけなら `FullscreenPass`
5. ぼかし自体を別のエフェクトでも使いたいなら `SeparableBlurPass`

## 1. ポストプロセスをアプリへ組み込む共通フロー

まず覚えるべきなのは、ポストプロセス処理は `app.space.draw(app.eye)` の代わりではなく、その描画先を切り替える仕組みだということです。`WebgApp` の通常描画をそのまま使うと、シーンは直接キャンバスへ出るため、あとから Bloom や被写界深度（DOF）を挟めません。そのため、ポストプロセス処理を使うサンプルでは `autoDrawScene: false` にしています。

最小構成は次の形です。

```js
import WebgApp from "./webg/WebgApp.js";
import FullscreenPass from "./webg/FullscreenPass.js";
import BloomPass from "./webg/BloomPass.js";

const app = new WebgApp({
  document,
  messageFontTexture: "./webg/font512.png",
  autoDrawScene: false
});
await app.init();

const bloom = new BloomPass(app.getGL(), {
  width: app.screen.getWidth(),
  height: app.screen.getHeight()
});
await bloom.ready;

const debugPass = new FullscreenPass(app.getGL(), {
  targetFormat: app.getGL().format
});
await debugPass.init();

app.start({
  onUpdate: ({ screen }) => {
    // 画面サイズが変わった場合は、オフスクリーンターゲットも追従させる
    bloom.resizeToScreen(screen);
  },
  onBeforeDraw: () => {
    // 3D シーンを画面ではなく sceneTarget へ描画する
    bloom.beginScene(app.screen, app.clearColor);
    app.space.draw(app.eye);
  },
  onAfterDraw3d: () => {
    // sceneTarget を入力として bloom 合成し、画面へ戻す
    bloom.render(app.screen, {
      clearColor: app.clearColor
    });

    // HUD を後で重ねる場合は深度バッファを空にしておく
    app.screen.clearDepthBuffer();
  }
});
```

このコードで重要なのは、`bloom.beginScene()` がシーン側の入口であり、`bloom.render()` がポストプロセス処理側の入口だということです。3D シーンを描画する責任は、あくまで `app.space.draw(app.eye)` にあり、BloomPass はその前後をつなぐ部品です。

## 2. `BloomPass` の使い方

### 背景

`BloomPass` は、シーンの中で明るい部分だけを抜き出し、それをぼかして、最後に元のシーンへ足し戻すヘルパーです。`webg/samples/bloom` は、この処理フローをそのまま見せる教材用サンプルになっています。

### 役割

`BloomPass` は内部で次のターゲットを持ちます。

1. `sceneTarget`
2. `extractTarget`
3. `extractHeatTarget`
4. `SeparableBlurPass` が持つぼかしターゲット

利用者が直接触るのは、主に次の API です。

| API                                    | 役割                               |
| -------------------------------------- | -------------------------------- |
| `new BloomPass(gpu, options)`          | Bloom パスを作る                      |
| `await bloom.ready`                    | GPU リソースの準備完了を待つ                 |
| `bloom.beginScene(screen, clearColor)` | `sceneTarget` への描画を開始する          |
| `bloom.render(screen, options)`        | extract / blur / composite を実行する |
| `bloom.resizeToScreen(screen)`         | ターゲットサイズを画面へ追従させる                |
| `bloom.setEnabled(flag)`               | ON / OFF                         |
| `bloom.setThreshold(value)`            | 抽出しきい値                           |
| `bloom.setExtractIntensity(value)`     | extract 強度                       |
| `bloom.setSoftKnee(value)`             | しきい値周辺の立ち上がり                     |
| `bloom.setBloomStrength(value)`        | 合成時の Bloom 強度                    |
| `bloom.setExposure(value)`             | 合成後の露出                           |
| `bloom.setToneMapMode(mode)`           | `0=off 1=reinhard 2=aces`        |
| `bloom.setBlurScale(value)`            | blur ターゲットの縮小率                   |
| `bloom.setBlurIterations(value)`       | blur の往復回数                       |
| `bloom.setBlurRadius(value)`           | blur 半径                          |

### 最小コード例

```js
import BloomPass from "./webg/BloomPass.js";

const bloom = new BloomPass(app.getGL(), {
  width: app.screen.getWidth(),
  height: app.screen.getHeight(),
  threshold: 0.68,
  extractIntensity: 1.0,
  softKnee: 0.35,
  bloomStrength: 1.15,
  exposure: 1.0,
  toneMapMode: 0,
  blurScale: 1.0,
  blurIterations: 2,
  blurRadius: 1.0
});
await bloom.ready;

app.start({
  onUpdate: ({ screen }) => {
    // resize ごとにオフスクリーンターゲットを作り直す必要はなく、
    // パス側の resize ヘルパーを呼べばよい
    bloom.resizeToScreen(screen);
  },
  onBeforeDraw: () => {
    // まずシーン全体をオフスクリーンターゲットへ描画する
    bloom.beginScene(app.screen, app.clearColor);
    app.space.draw(app.eye);
  },
  onAfterDraw3d: () => {
    // 次に bright extract -> blur -> composite をまとめて実行する
    bloom.render(app.screen, {
      clearColor: app.clearColor
    });
    app.screen.clearDepthBuffer();
  }
});
```

### デバッグビューを出す例

中間結果を見たいときは `FullscreenPass` を使います。`BloomPass` は中間ターゲットへアクセスする getter を持っているので、それをそのまま `debugPass.draw()` へ渡せます。

```js
import FullscreenPass from "./webg/FullscreenPass.js";

const debugPass = new FullscreenPass(app.getGL(), {
  targetFormat: app.getGL().format
});
await debugPass.init();

let bloomView = "composite";

app.start({
  onUpdate: ({ screen }) => {
    bloom.resizeToScreen(screen);
  },
  onBeforeDraw: () => {
    bloom.beginScene(app.screen, app.clearColor);
    app.space.draw(app.eye);
  },
  onAfterDraw3d: () => {
    bloom.render(app.screen, {
      clearColor: app.clearColor
    });

    if (bloomView !== "composite") {
      const debugSource = bloomView === "scene"
        ? bloom.getSceneTarget()
        : bloomView === "extract"
          ? bloom.getExtractTarget()
          : bloomView === "extractHeat"
            ? bloom.getExtractHeatTarget()
            : bloomView === "blurA"
              ? bloom.getBlurTargetA()
              : bloom.getBlurTargetB();

      app.screen.beginPass({
        clearColor: app.clearColor,
        colorLoadOp: "clear",
        depthView: null
      });
      debugPass.draw(debugSource);
    }

    app.screen.clearDepthBuffer();
  }
});
```

この形にしておくと、処理自体は毎フレームそのまま実行しつつ、「最後に何を画面へ出すか」だけを切り替えられます。

### 使いどころ

specular highlight、emissive orb、ライト付き UI オブジェクトの glow を足したいときに向いています。`webg/samples/bloom` は `threshold`、`softKnee`、`extractIntensity`、`blurRadius`、`blurIterations`、`exposure` の意味を視覚的に確かめるには最良の参照先です。

### 注意点

`BloomPass` は深度を後段で使いませんが、シーン自体は通常の 3D 描画なので、`sceneTarget` には深度 attachment が必要です。ここを単純なカラーのみのターゲットにすると、シーンの深度テストが崩れます。

## 3. `DofPass` の使い方

### 背景

`DofPass` は、シーンカラーと深度の両方を見ながら、合焦面だけをシャープに残し、それ以外を blur へ寄せるヘルパーです。`webg/samples/dof` は focus guide とデバッグビューを同時に出し、`focusDistance` と `focusRange` の意味が見えるようにしています。

### 役割

`DofPass` は内部で次のターゲットを持ちます。

1. `sceneTarget`。カラーとサンプリング可能な深度を持つ
2. `depthDebugTarget`
3. `focusDebugTarget`
4. `SeparableBlurPass` が持つぼかしターゲット

利用者がまず覚えるべき API は次のものです。

| API                                  | 役割                                          |
| ------------------------------------ | ------------------------------------------- |
| `new DofPass(gpu, options)`          | 被写界深度（DOF）パスを作る                             |
| `await dof.ready`                    | GPU リソースの準備完了を待つ                            |
| `dof.beginScene(screen, clearColor)` | `sceneTarget` への描画を開始する                     |
| `dof.render(screen, options)`        | depth debug / focus debug / composite を更新する |
| `dof.resizeToScreen(screen)`         | ターゲットサイズを画面へ追従させる                           |
| `dof.setEnabled(flag)`               | ON / OFF                                    |
| `dof.setFocusDistance(value)`        | 合焦面までの距離                                    |
| `dof.setFocusRange(value)`           | シャープに残す幅                                    |
| `dof.setMaxBlurMix(value)`           | ぼかしを混ぜる上限                                   |
| `dof.setBlurScale(value)`            | blur ターゲットの縮小率                              |
| `dof.setBlurIterations(value)`       | blur の往復回数                                  |
| `dof.setBlurRadius(value)`           | blur 半径                                     |

### 最小コード例

```js
import DofPass from "./webg/DofPass.js";

const dof = new DofPass(app.getGL(), {
  width: app.screen.getWidth(),
  height: app.screen.getHeight(),
  projectionNear: app.projectionNear,
  projectionFar: app.projectionFar,
  focusDistance: 34.0,
  focusRange: 6.0,
  maxBlurMix: 1.0,
  blurScale: 0.5,
  blurIterations: 2,
  blurRadius: 2.4
});
await dof.ready;

app.start({
  onUpdate: ({ screen }) => {
    // projection に対応する near / far は constructor で合わせておく
    dof.resizeToScreen(screen);
  },
  onBeforeDraw: () => {
    // まずシーンカラーと深度を sceneTarget へ描画する
    dof.beginScene(app.screen, app.clearColor);
    app.space.draw(app.eye);
  },
  onAfterDraw3d: () => {
    // blur 作成と depth-based composite をまとめて実行する
    dof.render(app.screen, {
      clearColor: app.clearColor
    });
    app.screen.clearDepthBuffer();
  }
});
```

### デバッグビューを出す例

`DofPass` は `depth` と `focusMask` をカラーターゲットへ出すヘルパーも内部で回しているので、デバッグ表示用の入力をそのまま画面へ出せます。

```js
import FullscreenPass from "./webg/FullscreenPass.js";

const debugPass = new FullscreenPass(app.getGL(), {
  targetFormat: app.getGL().format
});
await debugPass.init();

let dofView = "composite";

app.start({
  onUpdate: ({ screen }) => {
    dof.resizeToScreen(screen);
  },
  onBeforeDraw: () => {
    dof.beginScene(app.screen, app.clearColor);
    app.space.draw(app.eye);
  },
  onAfterDraw3d: () => {
    dof.render(app.screen, {
      clearColor: app.clearColor
    });

    if (dofView !== "composite") {
      const debugSource = dofView === "scene"
        ? dof.getSceneTarget()
        : dofView === "depth"
          ? dof.getDepthDebugTarget()
          : dofView === "focusMask"
            ? dof.getFocusDebugTarget()
            : dofView === "blurA"
              ? dof.getBlurTargetA()
              : dof.getBlurTargetB();

      app.screen.beginPass({
        clearColor: app.clearColor,
        colorLoadOp: "clear",
        depthView: null
      });
      debugPass.draw(debugSource);
    }

    app.screen.clearDepthBuffer();
  }
});
```

### 使いどころ

距離感を見せたいシーン、focus guide を含むデモ、近景と遠景をはっきり分けたいサンプルに向いています。`webg/samples/dof` は `focusDistance` と `focusRange` の違い、`maxBlurMix` と `blurRadius` の違い、`full / half` blur quality の違いを読みやすく確認できます。

### 注意点

`DofPass` は深度を後段シェーダーから読むので、projection の `near` / `far` がアプリ側のカメラと合っていないと、`focusDistance` の見え方がずれます。`projectionNear` と `projectionFar` は、`WebgApp` 側の値をそのまま渡すほうが安全です。

## 4. `VignettePass` の使い方

### 背景

`VignettePass` は最終色に対する最小のエフェクトです。Bloom や DOF のように複数段のターゲットを持たず、「1 枚のカラーテクスチャを読み、周辺だけを減衰させる」ことに集中しています。`webg/unittest/vignette` は、その最短の構成を確認するテストです。

### 役割

`VignettePass` は `FullscreenPass` を継承し、次のパラメータを持ちます。

| API                                    | 役割                                    |
| -------------------------------------- | ------------------------------------- |
| `new VignettePass(gpu, options)`       | Vignette パスを作る                        |
| `await vignettePass.init()`            | パイプラインを初期化する                          |
| `vignettePass.setCenter(x, y)`         | Vignette 中心の UV                       |
| `vignettePass.setRadius(value)`        | 外周半径                                  |
| `vignettePass.setSoftness(value)`      | 境界のやわらかさ                              |
| `vignettePass.setStrength(value)`      | 効きの強さ                                 |
| `vignettePass.setEnabled(flag)`        | ON / OFF                              |
| `vignettePass.setTint(r, g, b, a)`     | 周辺へ寄せる色                               |
| `vignettePass.render(screen, options)` | source を読み、キャンバスまたは destination へ描画する |

### 最小コード例

`VignettePass` 自体はオフスクリーンターゲットを持たないので、source 側は別に用意します。`webg/unittest/vignette` と同じ考え方で書くと、次のようになります。

```js
import FullscreenPass from "./webg/FullscreenPass.js";
import VignettePass from "./webg/VignettePass.js";

const sceneTarget = app.screen.createRenderTarget({
  label: "my-scene",
  format: app.getGL().format,
  hasDepth: true
});
await sceneTarget.ready;

const rawScenePass = new FullscreenPass(app.getGL());
await rawScenePass.init();

const vignettePass = new VignettePass(app.getGL(), {
  centerX: 0.5,
  centerY: 0.5,
  radius: 0.74,
  softness: 0.42,
  strength: 0.92,
  tint: [0.0, 0.0, 0.0, 1.0]
});
await vignettePass.init();

app.start({
  onUpdate: ({ screen }) => {
    sceneTarget.resizeToScreen(screen);
  },
  onBeforeDraw: () => {
    // sceneTarget へ普通の 3D シーンを描画する
    app.screen.clear(sceneTarget);
    app.space.draw(app.eye);
  },
  onAfterDraw3d: () => {
    // source テクスチャを読み、最終色へ vignette を掛けて戻す
    vignettePass.render(app.screen, {
      source: sceneTarget,
      clearColor: app.clearColor
    });
    app.screen.clearDepthBuffer();
  }
});
```

元のシーンと見比べたいだけなら、`vignettePass.render()` の代わりに `rawScenePass.draw(sceneTarget)` を呼べば比較できます。

### 使いどころ

最終色へ軽い雰囲気付けをしたいとき、画面中心を少し強調したいとき、ポストプロセス処理の最小確認をしたいときに向いています。シーンに依存した複雑な条件分岐が要らないなら、Bloom や DOF よりずっと軽く扱えます。

### 注意点

`VignettePass` はジオメトリや深度を理解しません。単に 1 枚のカラーテクスチャへ周辺減衰を掛けるだけなので、物体ごとに別扱いしたい場合には向きません。

## 5. `FullscreenPass` と `SeparableBlurPass` の位置付け

### `FullscreenPass`

`FullscreenPass` は、「1 枚のテクスチャを現在のパスへ出す」最小ヘルパーです。デバッグビュー、コピーパス、元のシーンの比較表示でよく使います。ポストプロセス処理を理解するときは、まず `FullscreenPass` を「中間ターゲットを画面で見るための表示器」と考えると整理しやすくなります。

### `SeparableBlurPass`

`SeparableBlurPass` は、横ぼかしと縦ぼかしをピンポンで回す共有ヘルパーです。`BloomPass` と `DofPass` はこのクラスを内部で使っています。通常の利用者は直接使わなくても困りませんが、「ぼかしだけを別のエフェクトでも使いたい」「blur ターゲットの縮小率や iterations の意味を知りたい」ときは、ここを読むと分かりやすくなります。

## 6. 利用者向けの導入パターン

### 6.1 最初の 1 本は `BloomPass` か `VignettePass`

ポストプロセス処理を初めて入れるなら、`BloomPass` か `VignettePass` のどちらか 1 本から始めるほうが安全です。どちらも「オフスクリーン -> フルスクリーン -> キャンバス」の形を追いやすく、深度を後段で読むところまでは要求しません。

### 6.2 被写界深度（DOF）はカメラ設定が固まってから

`DofPass` は見え方が `projectionNear` / `projectionFar` と強く結びつくため、カメラの視野角や `near` / `far` がまだ不安定な段階だと調整しづらくなります。シーンの基本カメラが固まってから入れるほうが扱いやすくなります。

### 6.3 デバッグビューを最初から用意する

ポストプロセス処理は、中間テクスチャを見られるだけで理解しやすさがかなり上がります。`BloomPass` なら `scene / extract / extractHeat / blurA / blurB`、`DofPass` なら `scene / depth / focusMask / blurA / blurB` を `FullscreenPass` で切り替えられるようにしておくと、パラメータ調整が速くなります。

## 7. よくあるつまずき

### 7.1 パスを作ったのに画面が変わらない

`autoDrawScene: false` にしておらず、シーンが直接キャンバスへ描画されている可能性があります。`beginScene()` と `render()` を描画フローへ組み込んでください。

### 7.2 resize 後に解像度がずれる

`onUpdate()` で `bloom.resizeToScreen(screen)` や `dof.resizeToScreen(screen)` を呼んでいない可能性があります。オフスクリーンターゲットは画面サイズへ追従させる必要があります。

### 7.3 被写界深度（DOF）が効かない

projection の `near` / `far` がカメラと合っていないか、あるいはデバッグビューで深度を確認せずにパラメータを動かしている可能性があります。まず `depth` と `focusMask` を確認すると、原因を切り分けやすくなります。

### 7.4 HUD が出なくなった

ポストプロセス処理の最終パスのあとで深度を空にしていない可能性があります。HUD やデバッグオーバーレイを重ねる構成なら、`app.screen.clearDepthBuffer()` を最後に入れてください。

### 7.5 `VignettePass` を作っただけで動くと考えてしまう

`VignettePass` は source テクスチャを自分で作りません。シーン自体をどこかの `RenderTarget` へ描画し、そのテクスチャを `render({ source })` へ渡す必要があります。
