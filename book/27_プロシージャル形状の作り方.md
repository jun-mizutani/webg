# プロシージャル形状の作り方

ローレベル API を学ぶ価値は、組み込み primitive を手で再実装することではありません。既製の形状では扱いにくい構造を、自分で設計してメッシュとして成立させられるところにあります。本章では、`icosphere`、メビウス帯、シェルピンスキー四面体、フラクタル地形を題材に、プロシージャル形状ごとに何を学ぶべきかを整理します。 

ここで最初に押さえたいのは、ローレベル API を触る目的は、単に「頂点を手で積む練習」をすることではない、という点です。組み込み primitive を少し組み替えただけでは作りにくい形を題材にしてはじめて、`Shape` の低水準経路が何のためにあるのかが見えやすくなります。つまり、この章の主眼は、`Shape` の API を暗記することではなく、「既製形状では扱いにくい構造を、自分でどう組み立てるか」を学ぶことにあります。 

また、この章では題材ごとに学べるポイントがはっきり異なります。`icosphere` では subdivision と頂点共有、メビウス帯ではパラメトリック曲面とシーム、シェルピンスキー四面体では再帰とシャープな面構成、フラクタル地形ではハイトフィールドと頂点グリッドが中心になります。つまり、どの題材も「プロシージャル形状」であることは共通していますが、見てほしい論点は同じではありません。その違いを意識しながら読むと、この章の教材群がずっと整理しやすくなります。 

さらに重要なのは、本文だけで完結させず、現在のサンプルと往復しながら読むことです。この章のコード例は教材サンプルと対応づけてあり、実際の見え方をサンプル側で確認できることに意味があります。つまり、「コードを読む → サンプルで見た目を確かめる → もう一度コードへ戻る」という往復を前提にした章です。ローレベル API の学習は、画面上で何が起きているかを見ながら進めるほうが理解しやすくなります。 

## どんな題材が教材になるか

円筒や UV シームを持つ球まで理解できたら、次は「組み込み形状を少し組み替えただけでは作りにくい形」を題材にすると学習効果が高くなります。候補としては次が有力です。`icosphere`、メビウス帯、シェルピンスキー四面体、経路に沿ったチューブ、ノイズベースのハイトフィールド地形、星形多角形の押し出しです。 

この中で、ローレベル API の次の教材として特に相性がよいのは `icosphere` とメビウス帯です。理由は、この 2 つが「頂点共有とトポロジー」「パラメトリック曲面とシーム」という、ローレベル形状構築の中でも特に基本になる 2 本柱をきれいに分けて見せてくれるためです。つまり、最初の 2 題材として非常にバランスがよくなります。 

## 最初に学ぶ 2 題材

### `icosphere`

`icosphere` がよい理由は、三角形 subdivision を学べること、頂点共有と中点キャッシュを学べること、法線を「面から計算する」方法と「半径方向から直接与える」方法を比較できること、planet mesh や LOD の入口になることです。`icosphere` のよいところは、単に球を作ることではなく、「同じエッジの中点を二重生成しない」というメッシュトポロジーの基本を非常に分かりやすく学べることです。つまり、共有トポロジーを持つメッシュをどう育てるかを知る教材として優秀です。 

### メビウス帯

メビウス帯がよい理由は、パラメトリック曲面の基本を学べること、帯の幅方向と周方向の 2 パラメータを扱えること、シームを閉じるために最後のリングを複製する理由が理解しやすいこと、「1 回ねじると表裏が連続する」特殊なトポロジーを体感できることです。メビウス帯のよいところは、「位置を直接列挙する」のではなく、「パラメータから位置を計算する」発想へ自然に進めることです。つまり、プロシージャル形状を数式ベースで考える入口としてとても優れています。 

## `icosphere` で学ぶ subdivision と共有トポロジー

`icosphere` は、正二十面体から始めて、各三角形を 4 分割しながら球面へ押し戻して作るメッシュです。通常の緯度経度球より三角形の偏りが少ないので、planet や滑らかな岩のようなメッシュに向いています。 

この題材で大事なのは、頂点を共有しながら面を増やすこと、同じエッジの midpoint を二重生成しないこと、法線を位置から直接与えると球として自然な見え方になることの 3 点です。つまりこのコードでは、「球を作る」こと自体よりも、「共有トポロジーを壊さず subdivision を進める」ことに注目して読むのが重要です。 

```js
import Shape from "./webg/Shape.js";

function buildIcosphere(shape, radius = 1.0, subdivisions = 2) {
  // 球では法線が「中心から外へ向く方向」と一致するため、
  // addTriangle() の面法線加算ではなく、位置から直接 normal を与える
  shape.setAutoCalcNormals(false);

  const vertices = [];
  let faces = [];

  const addSphereVertex = (x, y, z) => {
    const len = Math.hypot(x, y, z);
    const nx = x / len;
    const ny = y / len;
    const nz = z / len;

    // addVertex() は球面 UV を既定設定から自動計算してくれる
    const index = shape.addVertex(nx * radius, ny * radius, nz * radius) - 1;
    shape.setVertNormal(index, nx, ny, nz);
    vertices.push([nx * radius, ny * radius, nz * radius]);
    return index;
  };

  const t = (1.0 + Math.sqrt(5.0)) * 0.5;

  // 正二十面体の 12 頂点
  addSphereVertex(-1,  t,  0);
  addSphereVertex( 1,  t,  0);
  addSphereVertex(-1, -t,  0);
  addSphereVertex( 1, -t,  0);

  addSphereVertex( 0, -1,  t);
  addSphereVertex( 0,  1,  t);
  addSphereVertex( 0, -1, -t);
  addSphereVertex( 0,  1, -t);

  addSphereVertex( t,  0, -1);
  addSphereVertex( t,  0,  1);
  addSphereVertex(-t,  0, -1);
  addSphereVertex(-t,  0,  1);

  // 正二十面体の 20 面
  faces = [
    [0, 11, 5], [0, 5, 1], [0, 1, 7], [0, 7, 10], [0, 10, 11],
    [1, 5, 9], [5, 11, 4], [11, 10, 2], [10, 7, 6], [7, 1, 8],
    [3, 9, 4], [3, 4, 2], [3, 2, 6], [3, 6, 8], [3, 8, 9],
    [4, 9, 5], [2, 4, 11], [6, 2, 10], [8, 6, 7], [9, 8, 1]
  ];

  const getMidpoint = (cache, ia, ib) => {
    const key = ia < ib ? `${ia}:${ib}` : `${ib}:${ia}`;
    if (cache.has(key)) {
      return cache.get(key);
    }

    const a = vertices[ia];
    const b = vertices[ib];
    const mx = (a[0] + b[0]) * 0.5;
    const my = (a[1] + b[1]) * 0.5;
    const mz = (a[2] + b[2]) * 0.5;
    const index = addSphereVertex(mx, my, mz);
    cache.set(key, index);
    return index;
  };

  for (let level = 0; level < subdivisions; level++) {
    const nextFaces = [];
    const midpointCache = new Map();

    for (let i = 0; i < faces.length; i++) {
      const [a, b, c] = faces[i];
      const ab = getMidpoint(midpointCache, a, b);
      const bc = getMidpoint(midpointCache, b, c);
      const ca = getMidpoint(midpointCache, c, a);

      // 1 三角形を 4 三角形へ分割する
      nextFaces.push([a, ab, ca]);
      nextFaces.push([b, bc, ab]);
      nextFaces.push([c, ca, bc]);
      nextFaces.push([ab, bc, ca]);
    }

    faces = nextFaces;
  }

  for (let i = 0; i < faces.length; i++) {
    const [a, b, c] = faces[i];
    shape.addTriangle(a, b, c);
  }

  return {
    vertexCount: vertices.length,
    faceCount: faces.length
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildIcosphere(shape, 2.0, 2);
shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.52, 0.78, 0.96, 1.0]
});

console.log(info); // { vertexCount, faceCount }
```

このコードの見どころは、`addSphereVertex()` が位置を正規化してから半径を掛けること、法線は球の中心からの方向そのものを使うこと、`midpointCache` が無いと同じエッジの中点が何度も増えてトポロジーが壊れること、subdivision を 1 段上げるごとに三角形数は 4 倍になることです。ここで特に重要なのは `midpointCache` です。`icosphere` を教材として読む価値の多くは、まさにこの「共有すべき頂点を共有しないとメッシュが壊れる」という点にあります。 

## メビウス帯で学ぶパラメトリック曲面とシーム

メビウス帯は、帯を 1 回ひねって端同士をつないだ曲面です。組み込み形状を少し変形しただけでは作りにくく、パラメトリック曲面の教材としてとても優秀です。 

この題材で大事なのは、周方向 `u` と幅方向 `v` の 2 パラメータで面を作ること、頂点グリッドを先に作ってから面を張ること、シームを閉じるため最後のリングを複製すること、法線を面法線加算へ任せる設計も自然に使えることです。つまりこの題材では、「共有トポロジーをどう保つか」より、「パラメータから曲面をどう生成するか」と「シームをどう閉じるか」に注目して読むほうが理解しやすくなります。 

```js
import Shape from "./webg/Shape.js";

function buildMobiusStrip(shape, options = {}) {
  const radius = Number(options.radius ?? 2.2);
  const halfWidth = Number(options.halfWidth ?? 0.45);
  const uSegments = Math.max(3, Math.floor(options.uSegments ?? 96));
  const vSegments = Math.max(1, Math.floor(options.vSegments ?? 12));

  // 面法線加算 -> endShape() 正規化という標準経路を使う
  shape.setAutoCalcNormals(true);

  const grid = [];

  for (let iu = 0; iu <= uSegments; iu++) {
    const row = [];
    const u01 = iu / uSegments;
    const theta = u01 * Math.PI * 2.0;
    const halfTheta = theta * 0.5;

    for (let iv = 0; iv <= vSegments; iv++) {
      const v01 = iv / vSegments;
      const offset = (v01 - 0.5) * 2.0 * halfWidth;

      // webg は Y が上なので、輪の平面を XZ に置き、ひねり分だけ Y へ逃がす
      const x = (radius + offset * Math.cos(halfTheta)) * Math.cos(theta);
      const z = (radius + offset * Math.cos(halfTheta)) * Math.sin(theta);
      const y = offset * Math.sin(halfTheta);

      const index = shape.addVertexUV(x, y, z, u01, v01) - 1;
      row.push(index);
    }

    grid.push(row);
  }

  for (let iu = 0; iu < uSegments; iu++) {
    for (let iv = 0; iv < vSegments; iv++) {
      const p00 = grid[iu][iv];
      const p10 = grid[iu + 1][iv];
      const p11 = grid[iu + 1][iv + 1];
      const p01 = grid[iu][iv + 1];

      shape.addTriangle(p00, p10, p11);
      shape.addTriangle(p00, p11, p01);
    }
  }

  return {
    rows: grid.length,
    cols: grid[0]?.length ?? 0
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildMobiusStrip(shape, {
  radius: 2.4,
  halfWidth: 0.50,
  uSegments: 96,
  vSegments: 12
});

shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.94, 0.64, 0.42, 1.0]
});

console.log(info); // { rows, cols }
```

このコードの見どころは、頂点を「周方向の行」として持つと面張りが読みやすいこと、`iu <= uSegments` として最後のリングを複製することでシームを閉じていること、`u01` と `v01` をそのまま UV にしているため曲面パラメータとテクスチャ座標の対応が分かりやすいこと、`addTriangle()` の面法線加算をそのまま使って滑らかな見え方を得ていることです。ここで重要なのは、最後のリングを「つなぐ」のではなく「複製する」ことです。これはメビウス帯に限らず、多くのパラメトリック曲面でシームを扱う基本になります。 

### 両面表示と shader の局所調整

メビウス帯は「表と裏が連続する」特殊な面なので、両面を表示したい教材では陰影の扱いも題材になります。ここで注意したいのは、メビウス帯は非向き付け可能な曲面なので、帯全周で法線を完全に連続させることはできないという点です。 

それでも教材としては、少なくとも裏面側のライティングが不自然に反転しないほうが、帯全体を連続面として観察しやすくなります。このようなときは `SmoothShader` 本体を変更するのではなく、サンプル専用の派生クラスを 1 つ作り、その場でシェーダーを局所調整するほうが扱いやすくなります。つまり、ローレベル教材では「形状を組み立てること」だけでなく、「その題材に合わせてシェーダーを局所調整すること」も学習対象になります。 

`book/examples/27_02.html` では、実際に次のような `MobiusSmoothShader` を使っています。

```js
import SmoothShader from "../../webg/SmoothShader.js";

class MobiusSmoothShader extends SmoothShader {
  constructor(gpu) {
    super(gpu, {
      cullMode: "none"
    });

    // メビウス帯では裏面も同じ帯の続きとして見せたいので、
    // frontFacing に応じて法線向きをそろえる
    this.wgslSrc = this.wgslSrc.replace(
      "var nnormal = normalize(input.vNormal);",
      `let facing = select(-1.0, 1.0, input.frontFacing);
        var nnormal = normalize(input.vNormal) * facing;`
    );
  }
}
```

この例の重要な点は 2 つあります。ローレベルで shape を組み立てる教材でも、見え方の補正をシェーダーの派生クラスとして局所化できること。`SmoothShader` 本体を共通用途のまま保ちつつ、「この題材だけ両面の見え方をそろえたい」という要求をサンプル側で完結できることです。つまり、ローレベル API を使う意義は「頂点を直接積む」ことだけではありません。今回のメビウス帯のように、形状の性質に合わせてマテリアルやシェーダーの小さなカスタマイズを局所的に重ねるところまで含めて、教材として価値があります。 

## シェルピンスキー四面体で学ぶ再帰とシャープな面構成

フラクタルを題材にした例として、シェルピンスキー四面体はとてもよい教材です。正四面体を 4 つの小さな四面体へ分割し、その操作を再帰的に繰り返すことで、自己相似な 3D フラクタルを作れます。 

この題材の学習ポイントは、再帰でメッシュを増やすこと、1 つの大きな形状を複数の小さな形へ分けること、面ごとに頂点を複製しフラクタルの稜線をシャープに見せること、プロシージャルでなければ扱いにくい「段階的に育つ形」をコードで記述することの 4 つです。つまり、ここでは `icosphere` のように「共有を増やして整える」のではなく、「あえて面を分けて自己相似な折れ目を強調する」ことが教材の中心になります。前の 2 題材と対比しながら読むと理解しやすくなります。 

```js
import Shape from "./webg/Shape.js";

function buildSierpinskiTetrahedron(shape, options = {}) {
  const radius = Number(options.radius ?? 2.2);
  const depth = Math.max(0, Math.floor(options.depth ?? 3));

  // 面ごとに頂点を分けてフラットな見え方を保ちたいので、
  // addTriangle() の面法線加算 -> endShape() 正規化経路を使う
  shape.setAutoCalcNormals(true);

  const scale = radius / Math.sqrt(3.0);
  const v0 = [ scale,  scale,  scale];
  const v1 = [-scale, -scale,  scale];
  const v2 = [-scale,  scale, -scale];
  const v3 = [ scale, -scale, -scale];

  let triangleCount = 0;

  const midpoint = (a, b) => ([
    (a[0] + b[0]) * 0.5,
    (a[1] + b[1]) * 0.5,
    (a[2] + b[2]) * 0.5
  ]);

  const pushFace = (a, b, c) => {
    // フラクタルの角をはっきり見せたいので、面ごとに頂点を複製する
    const p0 = shape.addVertexUV(a[0], a[1], a[2], 0.0, 0.0) - 1;
    const p1 = shape.addVertexUV(b[0], b[1], b[2], 1.0, 0.0) - 1;
    const p2 = shape.addVertexUV(c[0], c[1], c[2], 0.5, 1.0) - 1;
    shape.addTriangle(p0, p1, p2);
    triangleCount++;
  };

  const addTetraFaces = (a, b, c, d) => {
    pushFace(a, c, b);
    pushFace(a, b, d);
    pushFace(a, d, c);
    pushFace(b, c, d);
  };

  const recurse = (a, b, c, d, level) => {
    if (level <= 0) {
      addTetraFaces(a, b, c, d);
      return;
    }

    const ab = midpoint(a, b);
    const ac = midpoint(a, c);
    const ad = midpoint(a, d);
    const bc = midpoint(b, c);
    const bd = midpoint(b, d);
    const cd = midpoint(c, d);

    // 中央の tetra は抜き、4 つの corner tetra だけを再帰する
    recurse(a,  ab, ac, ad, level - 1);
    recurse(ab, b,  bc, bd, level - 1);
    recurse(ac, bc, c,  cd, level - 1);
    recurse(ad, bd, cd, d,  level - 1);
  };

  recurse(v0, v1, v2, v3, depth);

  return {
    depth,
    triangleCount
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildSierpinskiTetrahedron(shape, {
  radius: 2.6,
  depth: 3
});

shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.96, 0.78, 0.42, 1.0],
  ambient: 0.16,
  specular: 0.90,
  power: 36.0
});

console.log(info); // { depth, triangleCount }
```

このコードの見どころは、`recurse()` が 1 つの tetra を 4 つの小 tetra に分割していること、`midpoint()` だけでフラクタルの段を育てられること、面ごとに頂点を複製することで稜線が丸まらずフラクタルらしいシャープさを保てること、`depth` を 1 段増やすだけで三角形数が大きく増えるので再帰の計算量も体感しやすいことです。ここでとくに重要なのは、「共有を減らすこと」が見た目の意図と一致している点です。前の `icosphere` では共有を保つことが重要でしたが、この例では逆です。つまり、プロシージャル形状では「共有するかどうか」は常に見た目の目的と結びついています。 

## フラクタル地形で学ぶハイトフィールド

フラクタル地形のハイトフィールドは、2D の高さ配列から 3D 地形を作る題材です。これは `TileMap` のような「セルごとに段差を持つ盤面」とは少し違い、1 枚の連続したメッシュ曲面を作るローレベル教材として相性がよくなります。つまり、この題材では「盤面データ」ではなく「連続曲面としての地形」を学ぶことが中心になります。 

この題材の学習ポイントは、2D グリッドの頂点を先に並べること、フラクタルノイズで高さを決めること、グリッドから四角形を張り `addPlane()` で三角形化すること、頂点共有によって滑らかな地形法線を得ること、読み込み機能や primitive ではなくデータからメッシュを育てる感覚をつかむことの 5 つです。ここで大事なのは、「高さ関数」と「メッシュ化」が明確に分かれていることです。つまり、地形を学ぶときは「最初に数学やノイズで高さを作り、そのあとで `Shape` に変換する」という 2 段構成で読むと非常に分かりやすくなります。 

### `TileMap` との違い

`TileMap` は「セルデータが主役」であり、移動判定や terrain 種別も同じデータから扱います。一方、この例は「見た目の曲面メッシュをプロシージャルに組む」ことが主役です。つまり、盤面ゲームの基準データを作りたいなら `TileMap`、連続した地形メッシュをローレベルで作りたいなら `Shape`、という使い分けになります。ここを先に切り分けておくと、同じ「地形」でも何を教材にしているのかが見えやすくなります。 

```js
import Shape from "./webg/Shape.js";

function buildFractalTerrain(shape, options = {}) {
  const cols = Math.max(2, Math.floor(options.cols ?? 48));
  const rows = Math.max(2, Math.floor(options.rows ?? 48));
  const sizeX = Number(options.sizeX ?? 10.0);
  const sizeZ = Number(options.sizeZ ?? 10.0);
  const heightScale = Number(options.heightScale ?? 2.4);
  const octaves = Math.max(1, Math.floor(options.octaves ?? 5));
  const persistence = Number(options.persistence ?? 0.52);
  const lacunarity = Number(options.lacunarity ?? 2.0);
  const seed = Number(options.seed ?? 1337);

  // 地形は頂点共有で滑らかに見せたいので、自動法線計算を使う
  shape.setAutoCalcNormals(true);

  const vertexGrid = Array.from({ length: rows + 1 }, () => new Array(cols + 1).fill(0));

  const hashNoise = (ix, iz, localSeed = seed) => {
    let h = localSeed | 0;
    h ^= Math.imul(ix + 1, 374761393);
    h ^= Math.imul(iz + 1, 668265263);
    h = Math.imul(h ^ (h >>> 13), 1274126177);
    h ^= h >>> 16;
    return (h >>> 0) / 4294967295;
  };

  const lerp = (a, b, t) => a + (b - a) * t;
  const fade = (t) => t * t * (3.0 - 2.0 * t);

  const valueNoise2d = (x, z) => {
    const x0 = Math.floor(x);
    const z0 = Math.floor(z);
    const x1 = x0 + 1;
    const z1 = z0 + 1;
    const tx = x - x0;
    const tz = z - z0;

    const n00 = hashNoise(x0, z0);
    const n10 = hashNoise(x1, z0);
    const n01 = hashNoise(x0, z1);
    const n11 = hashNoise(x1, z1);

    const sx = fade(tx);
    const sz = fade(tz);
    const nx0 = lerp(n00, n10, sx);
    const nx1 = lerp(n01, n11, sx);
    return lerp(nx0, nx1, sz);
  };

  const fbm = (x, z) => {
    let amplitude = 1.0;
    let frequency = 1.0;
    let sum = 0.0;
    let norm = 0.0;

    for (let octave = 0; octave < octaves; octave++) {
      sum += valueNoise2d(x * frequency, z * frequency) * amplitude;
      norm += amplitude;
      amplitude *= persistence;
      frequency *= lacunarity;
    }

    return norm > 0.0 ? sum / norm : 0.0;
  };

  const radialFalloff = (u, v) => {
    const dx = u - 0.5;
    const dz = v - 0.5;
    const d = Math.sqrt(dx * dx + dz * dz) * 2.0;
    return Math.max(0.0, 1.0 - d * d * 0.55);
  };

  for (let row = 0; row <= rows; row++) {
    const v = row / rows;
    const z = (v - 0.5) * sizeZ;

    for (let col = 0; col <= cols; col++) {
      const u = col / cols;
      const x = (u - 0.5) * sizeX;

      // 低周波と高周波を混ぜて地形の大きな起伏と細部を両立させる
      const coarse = fbm(u * 3.0 + 1.7, v * 3.0 + 2.9);
      const detail = fbm(u * 10.0 + 7.1, v * 10.0 + 5.3);
      const falloff = radialFalloff(u, v);

      // 島状の地形に寄せるため、中心ほど高く、外周は下げる
      const h01 = Math.max(0.0, coarse * 0.78 + detail * 0.22);
      const lifted = Math.pow(h01, 1.55);
      const y = (lifted * falloff - 0.12) * heightScale;

      const index = shape.addVertexUV(x, y, z, u, v) - 1;
      vertexGrid[row][col] = index;
    }
  }

  for (let row = 0; row < rows; row++) {
    for (let col = 0; col < cols; col++) {
      const p00 = vertexGrid[row][col];
      const p10 = vertexGrid[row][col + 1];
      const p11 = vertexGrid[row + 1][col + 1];
      const p01 = vertexGrid[row + 1][col];

      // 四角形を 2 三角形へ分けるヘルパーを使う
      shape.addPlane([p00, p01, p11, p10]);
    }
  }

  return {
    rows,
    cols,
    vertexCount: (rows + 1) * (cols + 1),
    quadCount: rows * cols
  };
}

const shape = new Shape(screen.getGL());
shape.setShader(shader);

const info = buildFractalTerrain(shape, {
  cols: 56,
  rows: 56,
  sizeX: 12.0,
  sizeZ: 12.0,
  heightScale: 2.8,
  octaves: 5,
  persistence: 0.54,
  lacunarity: 2.0,
  seed: 2026
});

shape.endShape();
shape.setMaterial("smooth-shader", {
  has_bone: 0,
  use_texture: 0,
  color: [0.40, 0.72, 0.38, 1.0],
  ambient: 0.20,
  specular: 0.42,
  power: 18.0
});

console.log(info); // { rows, cols, vertexCount, quadCount }
```

このコードの見どころは、`valueNoise2d()` が格子点の値を補間するだけの小さなノイズなので仕組みを追いやすいこと、`fbm()` がオクターブを重ねてフラクタルらしい粗密を作っていること、頂点を `vertexGrid[row][col]` に保持しておくと後段の面張りがとても読みやすいこと、`addPlane()` を使うと四角形単位で曲面を考えやすいこと、頂点共有のまま `addTriangle()` の法線加算を使うため地形表面が自然に滑らかになることです。 

## どの順で学ぶとよいか

順番としては、まず `icosphere` のほうが学びやすくなります。`icosphere` ではトポロジーと subdivision を学び、メビウス帯ではパラメトリック曲面とシームを学びます。`icosphere` では「頂点共有が壊れるとメッシュも壊れる」ことが見え、メビウス帯では「頂点グリッドを先に作ると面張りが整理しやすい」ことが見えます。この 2 つを理解すると、次はスプラインに沿ったチューブ、星形多角形の押し出し、ノイズ地形、回転体曲面の一般化といった題材へ進みやすくなります。つまり、この章では `icosphere` とメビウス帯を「最初の 2 題材」として位置づけ、その後の題材はそこからの発展と考えると学習順序が見えやすくなります。 

## 注意点

まず、`endShape()` を忘れると描画が出ません。`Shape` だけではシーンに置かれていないこと、表示位置は `Shape` ではなく `Node` 側で決めることも、前章と同じです。さらに、`setVertex()` / `addVertex*()` の戻り値は頂点数なので、インデックスに使うときは通常 `- 1` します。ここを忘れると、教材コードの多くがずれて見えます。 

また、プロシージャルメッシュでは、「頂点を共有するか、面ごとに分けるか」で法線の見え方が変わります。`icosphere` では共有が重要で、シェルピンスキー四面体では逆に面の独立性が重要です。つまり、この章の題材がどれだけ複雑になっても、結局は「頂点をどう積むか」「面をどう張るか」「共有するか分けるか」「最後に `endShape()` するか」という基本へ戻ってきます。難しい題材ほど第26章の原則を思い出しながら読むほうが理解しやすくなります。 

## まとめ

この章で最も大事なのは、ローレベル API を触る目的を見失わないことです。`icosphere`、メビウス帯、シェルピンスキー四面体、フラクタル地形は、それぞれ見た目が面白いだけの題材ではありません。`icosphere` は subdivision と共有トポロジー、メビウス帯はパラメトリック曲面とシーム、シェルピンスキー四面体は再帰とシャープな面構成、フラクタル地形はハイトフィールドと頂点グリッドを学ぶための教材です。つまり、この章では「何を作るか」よりも「その題材で何を学ぶか」を意識して読むことが大切です。 

また、本文とサンプルを往復しながら読むことにも意味があります。コードだけでは分かりにくい形状のつながりや法線の見え方、shader の局所調整の効果は、実際の sample で見ることで理解しやすくなります。ローレベル API を学ぶ価値は、単に低い層を触ることではなく、「既製形状では得にくい構造を、自分で設計してメッシュとして成立させる」経験にあります。この章は、その入口として読むと最も活きます。
