# はじめに

`webg` は、Chrome、Safari、Firefoxといったブラウザが標準で持っている JavaScript、WebGPU API、WebAudio API を利用して 3D アプリケーションを構築するためのライブラリです。PC やスマートフォンのブラウザ上で高速に動作する 3D アプリの作成を目的として設計されており、キー入力やタッチ操作、ピンチ操作、効果音や BGM の再生、GLB アニメーションのインポート、パーティクル、被写界深度、フォグなど、3D 表現に必要となる機能を網羅しています。外部ライブラリや複雑なビルドツールを導入する必要はなく、実装に必要なコードと詳細な解説はすべて `webg` 自身に含まれています。そのため、学習目的での利用はもちろん、AI を活用した効率的なアプリ開発にも適しています。また、WebGPU による 3D 開発で不可欠な座標計算や GPU 制御といったローレベル（低レイヤー）なプログラミングから、多くの機能が抽象化された `WebgApp` クラスによる高水準（高レイヤー）なプログラミングまで、柔軟に使い分けることが可能です。

![circular_breaker](fig01_01_circular_breaker.jpg)

`webg` では、低レイヤーと高レイヤーの両方を扱うことができます。そのため、学習の初期段階で重要なのはメソッド名を暗記することではなく、「どのような用途に向いたライブラリなのか」「どの文書から読み進めれば理解が早いのか」「サンプルと unittest をどう使い分ければよいのか」といった、学習の指針をつかむことです。

本章では、`webg` の全体像、適した用途、プロジェクト内のファイル配置、標準的な導入手順、サンプルの活用方法、問題調査の考え方、そして現時点での未実装範囲について解説します。詳細なメソッド一覧やクラス単位の仕様は付録B「API一覧」に譲り、本章では「どこから入り、どう読み進めるべきか」を判断するためのガイドとして構成しています。

## `webg` が提供する価値と設計思想

WebGPU を用いた 3D 実装では、描画処理が完成した後、アプリケーションの規模が大きくなるにつれて、シーングラフ、アセット管理、カメラ制御、入力処理、診断情報、音声再生といった周辺機能が必要になります。特に AI と共同で実装を進める場合、「どの機能がどこに定義されているか」「現在の挙動を理解するために何を読めばよいか」が曖昧であると、調査や修正の効率が著しく低下します。

`webg` はこの問題を解決するため、3D アプリケーションを構成する各レイヤーをリポジトリ内で明確に分離し、サンプルコードと解説文書を対にして読める構造を採用しています。巨大なエンジンをブラックボックスとして利用するのではなく、プロジェクトに必要な部分を順に読み解き、組み上げていくスタイルを重視しています。

具体的に、`webg` は以下のような場面で真価を発揮します。

* WebGPU を使った 3D アプリケーションの内部構造を理解しながら構築したいとき
* サンプル実装とドキュメントを往復しながら効率的に学びたいとき
* 診断情報、HUD、タッチ入力、音声処理までを単一のリポジトリで完結させたいとき
* 生成AIに 3D アプリ開発の支援をさせたいとき

学習にあたっては、まずいくつかの基本視点をそろえておくことが重要です。`webg/samples/` に用意されたサンプルは単なるデモではなく、実装手法を示す「教材」であること。`Touch`、`Diagnostics`、`Message`、`UIPanel` などの補助機能もアプリケーションの不可欠な構成要素として組み込まれていること。そして、3D の定義については第3章「3Dグラフィックスの基礎」に準拠すること。特にキーボードとタッチ操作を統合的に扱う場合は、第16章「タッチ機能と入力」を参考にしてください。

## 学習のロードマップと本書の歩き方

`webg` の最大の特徴は、3D の基礎からアプリケーション構成までが一つのリポジトリで完結している点です。描画、シーングラフ、アセット、アニメーション、HUD、入力、診断情報、音声といった要素を、外部資料をあちこち参照することなく系統立てて確認できます。

本書では、まず実行環境を整備し、3D の基礎と最小限の描画フローを押さえた後、`WebgApp`、アセット、アニメーション、TileMap、UI、診断情報、ポストプロセスへと段階的に進む構成をとっています。最初から API 一覧を参照するよりも、この順序に従うほうが学習効率は格段に高まります。

推奨される学習順序は以下の通りです。

1. 第2章「インストールと実行環境」で実行環境を整備する
2. 第3章「3Dグラフィックスの基礎」で座標系と回転の概念を理解する
3. 第4章「WebGPUとwebgの最小描画」で基本的な描画フローをつかむ
4. 第5章「WebgAppによるアプリ構成」で標準的なアプリケーション構造を学ぶ
5. `webg/samples/index.html` から `low_level`、`high_level`、`scene` のサンプルを確認する
6. 生成AIを利用して開発する場合は、最初に付録A「生成AIの皆さまへ」を AI に読み込ませる
7. その後、目的に応じて必要な章や付録へ進む

## プロジェクトの構成

`webg` を読み解く際は、まずファイル配置と役割を把握してください。

* `webg/`（コアライブラリ）: 
  `Screen`、`Space`、`Node`、`Shape`、`WebgApp`、`ModelBuilder`、`SceneLoader` など、ライブラリの本体となる実装が含まれています。
* `webg/samples/`（教材サンプル）: 
  機能確認とデモを兼ねたサンプル群です。各サンプルは `main.js`（実装）と `*.txt`（解説）で構成されており、`webg/samples/index.html` がその目次となっています。
* `webg/unittest/`（局所検証）: 
  特定の機能を切り出して検証するためのテストアプリ集です。問題の切り分けや最小単位での動作確認に適しており、`webg/unittest/index.html` からアクセスできます。

このように、ライブラリ本体、教材としてのサンプル、検証用の unittest を明確に分けることで、目的に応じた効率的な調査が可能です。

## `webg` で実現できること

`webg` は、WebGPU 上で 3D アプリケーションを構築するための基盤を包括的に備えています。

* 描画基盤: `Screen`、`Shader`、`Shape`、`Space`、`Node`、`RenderTarget`
* アプリ入口: `WebgApp`
* アセット・ローダー: `ModelAsset`、`ModelLoader`、`ModelBuilder`、`SceneAsset`、`SceneLoader`
* アニメーション: `Animation`、`Action`、`AnimationState`、`Skeleton`
* カメラ・入力: `EyeRig`、`Touch`、`InputController`
* UI・診断情報: `Message`、`DialogueOverlay`、`UIPanel`、`FixedFormatPanel`、`Diagnostics`、`DebugProbe`
* 音声: `AudioSynth`、`GameAudioSynth`
* ポストプロセス: `FullscreenPass`、`BloomPass`、`DofPass`、`VignettePass`

単なる描画ライブラリに留まらず、アプリケーションとして不可欠な周辺機能までを視野に入れて設計されているため、低レイヤーな描画制御から `WebgApp` を用いた高レイヤーな構成までを、一つのリポジトリ内で完結させることができます。

## 標準的な使い始め方

`webg` には、アプローチ方法に合わせて 2 つの入口が用意されています。

### 1. ローレベル（低レイヤー）による最小構成の理解
`Screen`、`Space`、`Shape`、`Matrix`、`SmoothShader` を直接的に連携させ、描画の流れを最短距離で確認する方法です。`webg/samples/low_level` にこの構成のサンプルが用意されています。

```js
import Screen from "./webg/Screen.js";
import Space from "./webg/Space.js";
import Shape from "./webg/Shape.js";
import Primitive from "./webg/Primitive.js";
import Matrix from "./webg/Matrix.js";
import SmoothShader from "./webg/SmoothShader.js";

const screen = new Screen(document);
await screen.ready;
screen.setClearColor([0.1, 0.15, 0.1, 1.0]);

const shader = new SmoothShader(screen.getGL());
await shader.init();
Shape.prototype.shader = shader;

const projection = new Matrix();
const applyViewportLayout = () => {
  screen.resize(
    Math.max(1, Math.floor(window.innerWidth)),
    Math.max(1, Math.floor(window.innerHeight))
  );
  projection.makeProjectionMatrix(
    0.1,
    1000.0,
    screen.getRecommendedFov(55.0),
    screen.getAspect()
  );
  shader.setProjectionMatrix(projection);
};
applyViewportLayout();
window.addEventListener("resize", applyViewportLayout);
window.addEventListener("orientationchange", applyViewportLayout);
shader.setLightPosition([120.0, 180.0, 140.0, 1.0]);

const space = new Space();
const eye = space.addNode(null, "eye");
eye.setPosition(0.0, 0.0, 8.0);

const shape = new Shape(screen.getGL());
shape.applyPrimitiveAsset(Primitive.cube(2.0, shape.getPrimitiveOptions()));
shape.endShape();
shape.setMaterial("smooth-shader", {
  use_texture: 0,
  color: [1.0, 0.5, 0.3, 1.0]
});

const obj = space.addNode(null, "obj");
obj.addShape(shape);

function loop() {
  obj.rotateY(0.8);
  obj.rotateX(0.4);
  screen.clear();
  space.draw(eye);
  screen.present();
  requestAnimationFrame(loop);
}
loop();
```

この例で注目すべきは、`await screen.ready` による初期化待機、プロジェクション行列の更新、`shape.endShape()` によるメッシュ確定、そして `clear` $\rightarrow$ `draw` $\rightarrow$ `present` という基本サイクルです。コアの仕組みを深く理解したい場合は、この最小構成から読み解くことを推奨します。

### 2. `WebgApp` による標準的なアプリ構成
`WebgApp` を利用して、カメラリグ、入力、HUD、診断情報などの補助機能をまとめて初期化する方法です。実用的なアプリケーション開発ではこちらが主流となります。`webg/samples/high_level` に最小例が用意されています。

```js
import WebgApp from "./webg/WebgApp.js";
import Shape from "./webg/Shape.js";
import Primitive from "./webg/Primitive.js";
import EyeRig from "./webg/EyeRig.js";

const CAMERA_CONFIG = {
  target: [0.0, 0.0, 0.0],
  distance: 8.0,
  yaw: 0.0,
  pitch: 0.0,
  bank: 0.0
};

const app = new WebgApp({
  document,
  messageFontTexture: "./webg/font512.png",
  clearColor: [0.1, 0.15, 0.1, 1.0],
  camera: CAMERA_CONFIG
});

await app.init();

const shape = new Shape(app.getGL());
shape.applyPrimitiveAsset(Primitive.cube(2.0, shape.getPrimitiveOptions()));
shape.endShape();
shape.setMaterial("smooth-shader", {
  use_texture: 0,
  color: [1.0, 0.5, 0.3, 1.0]
});

const obj = app.space.addNode(null, "obj");
obj.addShape(shape);

const orbit = new EyeRig(app.cameraRig, app.cameraRod, app.eye, {
  document,
  element: app.screen.canvas,
  input: app.input,
  type: "orbit",
  orbit: {
    target: CAMERA_CONFIG.target,
    distance: CAMERA_CONFIG.distance,
    yaw: CAMERA_CONFIG.yaw,
    pitch: CAMERA_CONFIG.pitch,
    minDistance: 4.0,
    maxDistance: 18.0,
    wheelZoomStep: 1.0
  }
});
orbit.attachPointer();

app.setGuideLines([
  "Drag: orbit",
  "2-finger drag: pan",
  "Pinch / wheel: zoom",
  "Arrow keys: orbit",
  "[ / ]: zoom"
], {
  anchor: "bottom-left",
  x: 0,
  y: -2
});

app.setStatusLines([
  "high_level sample",
  "WebgApp + EyeRig"
], {
  anchor: "top-left",
  x: 0,
  y: 0
});

app.start({
  onUpdate: ({ deltaSec }) => {
    orbit.update(deltaSec);
    obj.rotateY(0.8);
    obj.rotateX(0.4);
  }
});
```

この構成では、カメラ操作や操作ガイド、ステータス表示などを容易に組み込めます。マウス操作だけでなくタッチ操作を早期に検証したい場合は、こちらの方法が適しています。

## サンプルの活用方法

サンプルを確認する際は、`webg/samples/index.html` の一覧だけでなく、対応する `*.txt`（解説）と `main.js`（実装）をセットで読み込んでください。これにより、サンプルは単なる見本ではなく、具体的な実装手法を学ぶ教材として機能します。

特におすすめのサンプルは以下の通りです。

* `low_level`: 最小描画フローを即座に実行・確認できるサンプル
* `high_level`: `WebgApp` と `EyeRig` を用いた標準構成の最小例
* `scene`: Scene JSON、HUD、モデル読み込み、入力を統合的に確認できる入口
* `shapes`: primitive、texture、normal、wireframe の描画比較
* `gltf_loader`: glTF / `ModelAsset` / static bake の運用方法
* `sound`: `GameAudioSynth` と診断情報の連携確認
* `bloom`: オフスクリーンシーンと bloom 合成の基礎
* `dof`: 被写界深度（Depth of Field）とデバッグ表示の基礎

より規模のあるアプリケーション例としては `circular_breaker`、TileMap を活用した実践例としては `tile_sim` が非常に有用です。

なお、本ライブラリでは回転の定義として `head / pitch / bank` という用語を使用しています。一般的な 3D 用語における `yaw / pitch / roll` とそれぞれ対応しています。`EyeRig`、`CoordinateSystem`、`Animation` を読み解く際は、この対応関係を前提としてください。座標系、回転軸、テクスチャ座標、行列、クォータニオンの基礎事項は、第3章「3Dグラフィックスの基礎」にまとめています。

## 問題調査の手法と現状の限界

3D アプリケーションの開発では、描画結果だけでなく、「なぜその結果になったか」という根拠を追跡できることが重要です。`webg` では、ブラウザのコンソール出力のみに依存せず、画面内の HUD やパネルに情報を表示させ、必要に応じて診断情報レポートや probe を出力する手法を標準としています。後の章で解説する `Diagnostics`、`DebugDock`、`FixedFormatPanel` は、この「可視化によるデバッグ」という思想に基づいています。

一方で、現時点の `webg` には未実装の機能もあります。PBR（物理ベースレンダリング）、シャドウマップ、環境マップ、および高機能なエディターなどは未対応です。したがって、本ライブラリを選択する際は、巨大なエンジンをブラックボックスとして利用することよりも、実装されているレイヤーを読み解きながら、必要な機能を自前で組み立てていくプロセスに価値を見出していただけるかどうかが判断基準となります。

## まとめ

本章で最も重要な点は、`webg` を単なる API 集としてではなく、WebGPU の低レイヤーな描画から `WebgApp` による高レイヤーな構成までを体系的に学べるフレームワークとして捉えることです。

学習の入り口としては、第2章で環境を整え、第3章で座標系と回転を理解し、第4章で最小描画を確認し、第5章で `WebgApp` の標準構成へ進むルートが最もスムーズです。その上で、サンプルを「統合例」、unittest を「局所検証」として使い分け、必要に応じて各章の詳細へと進んでください。

本書が、WebGPU を用いた 3D アプリケーション開発の地図として役立つことを願っています。
