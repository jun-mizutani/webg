# あとがき

ここまでお読みいただき、ありがとうございました。

本書は、JavaScript と WebGPU を用いて 3D アプリケーションを構築するためのライブラリ `webg` を題材に、その使い方だけでなく、考え方や内部構造まで含めて一冊でたどれるようにすることを目指して書きました。最初の 3D 表示から始まり、`WebgApp` による標準的なアプリ構成、モデルやシーンの読み込み、アニメーション、UI、TileMap、そしてローレベル API やスキニングの仕組みに至るまで、かなり広い範囲を扱っています。そのため、読者によっては「少し踏み込みすぎている」と感じられた箇所もあったかもしれません。

しかし、本書では、あえて表面的な使い方だけで終わらせないようにしました。3D アプリケーションは、動いているあいだは分かったように見えても、少し表示が崩れたり、入力が思ったようにつながらなかったり、モデルやアニメーションの挙動が想定とずれたりすると、急に見通しが悪くなります。そのときに役立つのは、「この API を呼べばよい」という断片的な知識よりも、「いま自分はどの層の問題を見ているのか」を判断できる見取り図です。本書が一貫して目指したのは、その見取り図を読者に渡すことでした。

`webg` は、高水準 API だけで完結する巨大なブラックボックスとして設計したライブラリではありません。`WebgApp` のようにすぐ使える入口を持ちながらも、必要であれば `Shape`、`Shader`、`Matrix`、`Quat`、WGSL、スキニングの原理まで降りていけるようにしています。これは便利さだけを優先した設計ではありません。ライブラリを使う人が、必要になったときに内部まで追えること、そして「なぜそう動いているのか」を確認できることを大切にしているからです。本書もまた、その考え方に合わせて、入口から内部へ少しずつ降りていける構成を意識しました。

また、本書は単なる API リファレンスではなく、サンプル、本文、コア実装が互いに行き来できる資料でありたいと考えました。サンプルは単なる見本ではなく教材であり、本文は単なる説明ではなく読書順の案内であり、実装は単なる内部ではなく必要なときに参照できる一次資料です。読者がそのどれか一つだけを見て終わるのではなく、必要に応じて三者を往復できるようにしておくことが、`webg` のようなライブラリには重要だと考えています。

近年では、人間だけでなく、生成 AI と一緒にプログラムを作る機会も増えてきました。その意味でも、技術文書に求められるものは少しずつ変わってきているように感じます。単に断片的な使い方を並べるだけではなく、用語が一貫しており、層の境界が明確で、どの章がどの実装やどのサンプルに対応しているかが分かることが、以前よりもずっと重要になっています。本書が結果としてそのような形になっているとすれば、それは `webg` を人間が理解しやすくしたいという意図と、AI を含めた支援の道具が読み違えにくい形にしたいという意図が、同じ方向を向いていたからだと思います。

もちろん、本書に書かれている内容がすべて最終形というわけではありません。`webg` 自体もまだ発展の途中にありますし、未実装の機能や、今後よりよい形へ整理されていく部分もあります。したがって、本書は完成された巨大システムの解説書というより、現時点の `webg` をできるだけ正確に、できるだけ実用的に、できるだけ見通しよくまとめた記録に近いものです。もし将来 `webg` が変わっていくなら、本書もまた、それに合わせて育っていくべき文書なのだと思います。

この本を読んでくださった方の中には、最初から最後まで通読された方もいれば、必要な章だけを行き来しながら読まれた方もいるはずです。どちらの読み方でも構いません。本書が、`webg` を使って何かを作ろうとするときの入口になり、詰まったときの参照先になり、内部まで確かめたくなったときの足場になっていれば、著者としてそれ以上にうれしいことはありません。

最後に、本書の内容は、サンプルや実装を読みながら少しずつ整えられてきたものです。もし本書を通して `webg` に興味を持ち、実際に何かを作ってみようと思っていただけたなら、それがいちばんの成果です。立方体を一つ回すところからでも、glb を読み込んでキャラクターを動かすところからでも、TileMap の上で actor を歩かせるところからでも構いません。小さな一歩でも、実際に手を動かしてみると、この本の内容はきっと別の見え方をしてくるはずです。

本書が、その最初の一歩と、その先の試行錯誤の両方に、少しでも役立つことを願っています。

ありがとうございました。
